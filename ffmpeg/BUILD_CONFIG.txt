FFmpeg Build Configuration for ConvertSave
===========================================

This document describes how FFmpeg is compiled for distribution with ConvertSave.
All builds use LGPL-only configuration (no --enable-gpl, no --enable-nonfree).
Static linking is used on all platforms (--enable-static --disable-shared).

ConvertSave invokes FFmpeg as a standalone CLI process, not by linking against
FFmpeg's libraries. The LGPL dynamic-linking requirement applies only when LGPL
code is embedded into a proprietary binary. Since ConvertSave does not link to
FFmpeg libraries, distributing a statically-linked FFmpeg binary is compliant.

Source: https://github.com/FFmpeg/FFmpeg
License: LGPLv2.1+

Note: --enable-libwavpack is not available in FFmpeg 7.x (the option was removed).
FFmpeg's built-in/native WavPack decoder is still included automatically.


macOS (x86_64 and arm64)
------------------------

./configure \
    --enable-static \
    --disable-shared \
    --pkg-config-flags="--static" \
    --enable-version3 \
    --enable-videotoolbox \
    --enable-avfoundation \
    --enable-audiotoolbox \
    --enable-libvpx \
    --enable-libaom \
    --enable-librav1e \
    --enable-libwebp \
    --enable-libopus \
    --enable-libvorbis \
    --enable-libmp3lame \
    --enable-libzimg \
    --enable-libxml2 \
    --enable-zlib \
    --enable-lzma

Dependencies (installed via Homebrew):
  libvpx, aom, rav1e, webp, opus, libvorbis, lame, zimg, libxml2


Windows (x86_64)
----------------

./configure \
    --enable-static \
    --disable-shared \
    --pkg-config-flags="--static" \
    --enable-version3 \
    --enable-mediafoundation \
    --enable-libvpx \
    --enable-libaom \
    --enable-libwebp \
    --enable-libopus \
    --enable-libvorbis \
    --enable-libmp3lame \
    --enable-libzimg \
    --enable-zlib \
    --extra-ldflags="-static"

Dependencies (installed via MSYS2 pacman):
  mingw-w64-x86_64-libvpx, mingw-w64-x86_64-aom,
  mingw-w64-x86_64-libwebp, mingw-w64-x86_64-opus,
  mingw-w64-x86_64-libvorbis, mingw-w64-x86_64-lame,
  mingw-w64-x86_64-zimg


Linux (x86_64)
--------------

./configure \
    --enable-static \
    --disable-shared \
    --enable-version3 \
    --enable-libvpx \
    --enable-libaom \
    --enable-libwebp \
    --enable-libopus \
    --enable-libvorbis \
    --enable-libmp3lame \
    --enable-libzimg \
    --enable-zlib \
    --enable-lzma

Note: Linux builds statically link FFmpeg's own libraries and codec libraries,
but dynamically link system libraries (glibc, etc.) since fully static builds
on Linux are problematic with glibc's NSS/DNS subsystems.

Dependencies (installed via apt):
  libvpx-dev, libaom-dev, libwebp-dev, libopus-dev,
  libvorbis-dev, libmp3lame-dev,
  libzimg-dev, zlib1g-dev, liblzma-dev, nasm


Explicitly NOT enabled (GPL/non-free):
  --enable-gpl
  --enable-nonfree
  --enable-libx264
  --enable-libx265
  --enable-libfdk-aac
